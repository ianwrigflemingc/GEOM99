Building a Web Map

The map displayed in the final solution was configured using AGOL Map Viewer (beta). The following steps were followed to cartographically render the forest resources information.

1) Add Data: The forest processing facility point layer and the forest management unit (FMU) polygon layer containing the Ontario wood report data were added to the map. To facilitate rendering, the point layer was stacked on top of the polygon layer.

2) Select Base Map: A simple light-gray base map was chosen to add spatial context to the map without distracting from the focus of the application.

3) Set Symbology: The FMU layer was displayed as a solid outline to show the bounds of the management unit. Each processing facility was displayed with a unique colour based on the facility type, and then further categorised by shape to help users differentiate between the sites.

4) Edit Fields: Many of the field names associated with the data layers were coded, so the field names for both layers were edited into user-friendly titles so that they can be easily interpreted.

5) Configure Pop-Ups: Fields to be included in the pop-up windows for both layers were selected based on what information was relevant to the end-user. For the facility layer we chose to include information such as the facility type, owner, identification, location, and operating status. For the FMU layer, we chose to include both management information such as the FMU name, area, and licence holder, and information on the available wood resources for the area. For the FMU pop-ups, we also included a pie chart to display the total merchantable and unmerchantable timber resources and text boxes to further explain the information displayed.
